# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

Only the latest minor release receives security patches. Users are encouraged to upgrade promptly.

## Reporting a Vulnerability

If you discover a security vulnerability in `contract-governor`, please report it responsibly. **Do not open a public GitHub issue.**

### Preferred: GitHub Security Advisories

The preferred method is to use [GitHub Security Advisories](https://github.com/evanerwee/contract-governor/security/advisories/new). This allows private discussion and coordinated disclosure.

### Fallback: Email

If you are unable to use GitHub Security Advisories, email the maintainer directly:

**Evan Erwee** — [evan@erwee.com](mailto:evan@erwee.com)

Please encrypt sensitive details with the maintainer's PGP key if available.

## Expected Response Timeline

| Stage                  | Timeframe         |
| ---------------------- | ----------------- |
| Acknowledgment         | Within 48 hours   |
| Initial assessment     | Within 7 days     |
| Fix development        | Within 30 days    |
| Coordinated disclosure | Within 90 days    |

If the issue is particularly complex, the maintainer will communicate revised timelines within the initial assessment window.

## What to Include in Your Report

- A clear description of the vulnerability
- Steps to reproduce (minimal proof of concept preferred)
- Affected version(s)
- Potential impact assessment
- Any suggested fix or mitigation (optional)

## What NOT to Do

- **Do not** disclose the vulnerability publicly before a fix is available
- **Do not** exploit the vulnerability beyond what is necessary to demonstrate it
- **Do not** access or modify other users' data
- **Do not** perform denial-of-service testing against production systems

## Recognition

We appreciate responsible disclosure. Contributors who report valid security issues will be credited in the release notes (unless they prefer to remain anonymous).
